# 모바일 최적화 계획 (sample-report.html)

## 목적
PC 기준으로 제작된 `public/reports/sample-report.html`를 **모바일 화면에서도 읽기 쉬운 보고서**로 제공한다.

## 범위
- 대상: `public/reports/sample-report.html`
- **HTML 구조 변경 없음** (CSS만 변경)
- **프린트 레이아웃 유지** (`@media print`는 그대로)
- 콘텐츠는 동일하게 유지 (모바일 축약/숨김 없음)

## 핵심 제약
- Tailwind CDN 기반의 유틸리티 클래스가 이미 적용되어 있음
- HTML 변경 불가 → **CSS 오버라이드 방식**으로만 반응형 처리
- 프린트 디자인 영향 금지

## 접근 방식 (하이브리드)
1) **전역 모바일 스케일**: 타이포/패딩/간격을 축소하여 기본 가독성 확보
2) **전역 그리드 1열 전환**: 일반 그리드는 모바일에서 자동 스택
3) **섹션별 예외 처리**: 테이블/히트맵/네트워크/차트 등 특이 블록에만 추가 규칙

## 적용 위치
- `sample-report.html` 내부 `<style>` 블록에 다음을 추가:
  - `@media screen and (max-width: 640px) { ... }`
  - `@media print`와 충돌하지 않게 screen 전용으로 분리

## 전역 모바일 규칙 (요약)
- `body`, `main`, `.card-shadow`의 패딩 축소
- 헤더의 `flex`를 세로 스택으로 전환, 칩은 줄바꿈 허용
- 큰 타이틀(`text-3xl`, `text-2xl`)은 모바일 크기로 축소
- `main .grid`는 기본적으로 1열로 덮어쓰기
- 버튼/칩 간격 최소 8px 확보

## 섹션별 조정 포인트
- **Executive Summary(3카드)**: 1열 스택 + 카드 간 간격 확보
- **리스크 테이블 / 벤치마크 테이블**:
  - `display:block; overflow-x:auto`로 수평 스크롤 허용
  - 헤더/셀 폰트 크기 축소
- **공급사 포트폴리오(3분할 차트)**: 1열 스택 + 차트 높이 축소
- **카테고리 도넛 + 인사이트**: `flex` → 세로, 도넛 크기 축소
- **히트맵(고정 그리드)**: 가로 스크롤 유지, 패딩/폰트 축소
- **페르소나 5열 카드**: 2열(필요시 1열)로 전환
- **온톨로지 네트워크**:
  - 좌우 2열 `flex` → 세로 스택
  - 네트워크 컨테이너 크기 축소(예: 300px → 220px)
- **프리미엄 AI 섹션**:
  - 헤더 세로 스택
  - 알고리즘 흐름 UI는 `flex-wrap` 허용
  - 차트 영역 높이 축소

## 검증 체크리스트
1) 모든 섹션이 1열로 자연스럽게 스택되는가
2) **가로 스크롤은 필요한 영역에만** 나타나는가
3) 텍스트 최소 12px 수준에서 읽기 가능한가
4) 차트/캔버스가 뷰포트를 넘지 않는가
5) 탭/스크롤 시 오작동이나 겹침이 없는가
6) `@media print` 출력이 기존과 동일한가

## 리스크 및 대응
- **전역 유틸리티 오버라이드 부작용** → 규칙 범위를 `main` 하위로 제한, 섹션별 예외 추가
- **Chart.js 리사이징 오류** → 캔버스 `max-width:100%`와 부모 높이 지정
- **가로 스크롤 인지 부족** → 설명 텍스트/여백으로 스크롤 힌트 제공

## 작업 순서
1) `@media screen and (max-width: 640px)` 추가
2) 전역 규칙 적용 (타이포/패딩/그리드)
3) 섹션별 예외 규칙 추가
4) 모바일 뷰포트 6종에서 시각 검증
5) 문제 섹션 보정

## 완료 기준 (Definition of Done)
- 모바일에서 **리스트/차트/테이블을 끊김 없이 스크롤**할 수 있음
- 핵심 텍스트가 1회 스크롤만으로 **읽을 수 있는 밀도**로 정리됨
- 데스크톱 및 프린트 레이아웃이 기존과 동일하게 유지됨
